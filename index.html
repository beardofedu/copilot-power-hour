Multi-Agent
Orchestration

Chaining multiple agents & tools in a single workflow

15 min

What is agent orchestration?

Copilot can run specialized agents in sequence or parallel — each handles a focused sub-task and passes context to the next.

User request
→
Explore agent
research
→
Edit agent
implement
→
Test agent
verify

Why it matters

  • Single prompts hit context limits on large tasks
  • Specialists outperform generalists on focused work
  • Independent work streams run in parallel

Mental model shift

  • One giant prompt → Research → Implement → Verify
  • Re-explain every time → Use #codebase
  • Watch Copilot work → Assign async, review later

Pattern 1 — Implicit orchestration

Describe a complex task in VS Code Chat. Copilot invokes tools automatically.

Refactor the auth module: find all usages, update them to
use the new JWT interface, then run the tests and fix failures.

Watch the tool calls unfold in the chat sidebar:

search_codebase
→
edit_file ×N
→
run_terminal
→
edit_file (fixes)
Show a multi-step task in VS Code Chat — watch the tool call chain in the sidebar

Pattern 2 — CLI chaining

Use gh copilot to build context, then feed it forward.

# Step 1 — understand the system
gh copilot explain "how does our rate limiter work"

# Step 2 — use that context in your next prompt
gh copilot suggest "add retry logic to the HTTP client \
  based on the rate limiter pattern"
💡 Shell aliases save time — add these to ~/.zshrc:
alias cops='gh copilot suggest'
alias cope='gh copilot explain'

Pattern 3 — Coding Agent (async)

Assign GitHub issues directly to Copilot. It opens a PR, runs CI, iterates on failures — without you watching.

# Assign an issue to the coding agent
gh issue edit 42 --add-assignee "@copilot"

# Or from GitHub.com: Assignees → Copilot

Write great issue briefs

  • Context & background
  • Specific constraints
  • Acceptance criteria
  • Files/modules to touch

Review the PR, not the process

  • Copilot explains its reasoning in the PR description
  • CI runs automatically
  • Comment on the PR to steer iterations
Assign a real issue to @copilot and show the resulting PR

CLI — Two commands, infinite use cases

# Install / verify
gh extension install github/gh-copilot
gh copilot --version

# suggest — "what command do I need?"
gh copilot suggest "find files modified in last 7 days larger than 10MB"

# explain — "what does this do?"
gh copilot explain "awk '{print $2}' | sort | uniq -c | sort -rn"

# Target a specific context
gh copilot suggest -t shell   # shell command
gh copilot suggest -t git     # git command
gh copilot suggest -t gh      # GitHub CLI command
Live terminal — suggest a complex git command, then explain a scary one-liner

Context references — use #

@workspace  "Find all API endpoints missing auth middleware"
#file:src/routes/users.ts  "Why is this returning 403?"
#codebase   "How is logging configured?"
#selection  "Refactor this to use async/await"
Use @workspace for a cross-file question, then switch to Edits mode to apply the fix

Chat participants you should know

@workspace

Full repo awareness — semantic search across all files

@github

PRs, issues, code search — requires GitHub sign-in

@terminal

Current terminal state, last command output, errors

@vscode

Settings, commands, extension questions

Pro tip: Combine them — @workspace #file:src/api.ts "explain this to a new team member"

User Settings

Configure once — apply to every project and chat session

8 min

User settings — configure once, use everywhere

Open: Cmd+Shift+P → "Open User Settings (JSON)"
Location (macOS): ~/Library/Application Support/Code/User/settings.json
Priority order (highest → lowest):
1. Inline instructions in the prompt
2. .github/copilot-instructions.md (repo-level)
3. User settings instructions array (global ← you control this)
Open User Settings JSON live and walk through the file structure

The settings that matter most

{
  // Inline suggestions
  "editor.inlineSuggest.enabled": true,
  "github.copilot.editor.enableAutoCompletions": true,

  // Chat behavior
  "github.copilot.chat.followUps": "always",
  "github.copilot.chat.edits.enabled": true,
  "github.copilot.chat.agent.runTasks": true,

  // Instruction files
  "github.copilot.chat.codeGeneration.useInstructionFiles": true,

  // Full codebase indexing
  "github.copilot.chat.indexing.enabled": true,

  // MCP
  "github.copilot.chat.mcp.enabled": true
}

Global instructions — the multiplier

Set team-wide defaults that apply to every chat session:

"github.copilot.chat.codeGeneration.instructions": [
  {
    "text": "We use conventional commits. Always suggest messages
             in the format: type(scope): description"
  },
  {
    "text": "Our stack: React 18, TypeScript strict, Vitest, Tailwind.
             Use these — not alternatives."
  },
  {
    "text": "Always add error handling. Prefer async/await over .then()."
  }
]
Add one instruction, ask Copilot to write a commit message — before & after

Q&A

3 minutes

Common questions

Is Copilot sending my code to the cloud?

Copilot uses code as context for suggestions. Enterprise plans include IP indemnity and data privacy controls. Check your org's policy.

How do I stop inline suggestions mid-typing?

Cmd+Shift+P → "Copilot: Toggle Inline Suggestions" — or set "github.copilot.editor.enableAutoCompletions": false

Individual vs Business vs Enterprise?

Individual: personal, basic features  ·  Business: org management, audit logs, policy controls  ·  Enterprise: fine-tuned models, Knowledge Bases, strict data controls

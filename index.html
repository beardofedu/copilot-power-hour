Multi-Agent Orchestration

Chaining multiple agents & tools in a single workflow

What is agent orchestration?

Copilot runs specialized agents in sequence or parallel — each handles a focused sub-task and passes context to the next.

User request
→
Explore
research
→
Edit
implement
→
Test
verify

Why it matters

  • Single prompts hit context limits on large tasks
  • Specialists outperform generalists on focused work
  • Independent streams run in parallel with /fleet

Mental model shift

  • One giant prompt → Research → Implement → Verify
  • Re-explain every time → Use #codebase
  • Watch Copilot work → Assign async, review later

Pattern 3 — Copilot Coding Agent (async, GitHub-native)

Assign GitHub issues to Copilot. It opens a PR, runs CI, and iterates on failures asynchronously.

# Assign from CLI
gh issue edit 42 --add-assignee "@copilot"

# Or from GitHub.com:
# Assignees → Copilot
Max runtime: 59 minutes. Available on all paid plans.

Write great issue briefs

  • Context & background
  • Constraints
  • Acceptance criteria
  • Files/modules

Review the PR, not the process

  • Reasoning appears in the PR description
  • Comment on the PR to steer iterations
Assign a real issue to @copilot and show the resulting PR

Pattern 4 — GitHub Agentic Workflows Public Preview

Define AI automations as Markdown — Copilot compiles them to hardened GitHub Actions workflows.

---
# .github/workflows/triage.workflow.md
on: issues
safe-outputs:
  - add-comment
max-ai-credits: 500
---
When a new issue is opened, analyze it,
add a label and triage comment.

Supported engines

Copilot (default) · Claude · Codex · Gemini

Safety model

  • Read-only by default
  • Write actions require safe-outputs
  • Threat scan before write ops

Personal repo = Copilot config layer

my-copilot-config/
├── .github/
│   ├── copilot-instructions.md   ← repo-wide
│   └── instructions/
│       └── api.instructions.md  ← path-specific
├── AGENTS.md      ← all agents (Copilot, Claude, Codex)
└── settings/
    └── vscode-settings.json
In the CLI, all instruction files combine rather than using priority-based fallbacks.

Instruction priority (high → low)

  1. Personal (GitHub.com chat settings)
  2. Path-specific .github/instructions/*.instructions.md
  3. Repo-wide .github/copilot-instructions.md
  4. AGENTS.md
  5. Org custom instructions (Business/Enterprise only)
# .github/copilot-instructions.md
You are a senior TypeScript engineer. Always:
- Prefer `const` over `let`
- Add JSDoc to exported functions
- Flag any `any` types with a comment

MCP — Reach outside the codebase

Install in VS Code (easiest)

Browse the GitHub MCP Registry from Extensions — no manual JSON needed.

Or add via User Settings JSON

{
  "github.copilot.chat.mcp.enabled": true,
  "mcp": {
    "servers": {
      "github": {
        "command": "npx",
        "args": ["-y", "@modelcontextprotocol/server-github"],
        "env": { "GITHUB_PERSONAL_ACCESS_TOKEN": "${env:GITHUB_TOKEN}" }
      } } } }
Install from MCP Registry in VS Code, then ask @github "summarize the last 5 merged PRs"

VS Code User Settings

Configure once — apply to every project and chat session

The settings that matter most

Open: Cmd+Shift+P → "Open User Settings (JSON)" · Lives at ~/Library/Application Support/Code/User/settings.json
{
  "editor.inlineSuggest.enabled": true,
  "github.copilot.editor.enableAutoCompletions": true,
  "github.copilot.chat.followUps": "always",
  "github.copilot.chat.edits.enabled": true,
  "github.copilot.chat.agent.runTasks": true,
  "github.copilot.chat.codeGeneration.useInstructionFiles": true,
  "github.copilot.chat.indexing.enabled": true,
  "github.copilot.chat.mcp.enabled": true
}
Priority order (high → low):
1. Inline prompt instructions
2. Path-specific .github/instructions/
3. .github/copilot-instructions.md
4. AGENTS.md
5. Org instructions (Business/Enterprise)
6. User settings instructions array (global)

Global instructions — the multiplier

Set team-wide defaults that apply to every chat session across all repos:

"github.copilot.chat.codeGeneration.instructions": [
  { "text": "We use conventional commits. Suggest: type(scope): description" },
  { "text": "Our stack: React 18, TypeScript strict, Vitest, Tailwind — use these." },
  { "text": "Always add error handling. Prefer async/await over .then()." }
]
Combine with .github/copilot-instructions.md in each repo for fine-grained control.
Add one instruction, ask Copilot to write a commit message — show the before & after

Tips, Tricks & Hidden Gems

Built-in ✨ moments & new features

Sparkle ✨ shortcuts

  • Commit messages — click ✨ in Source Control panel
  • PR descriptions — "Copilot" button when creating a PR
  • Terminal errors — click ✨ next to a failed command
  • Kebab ··· in chat — "Insert into Terminal" / "Insert at Cursor"

Auto model + BYOK

Set model to Auto — Copilot picks the best model per task (10% AI credit discount). Or bring your own API key for custom/private models.

Next edit suggestions

Predicts the next location you'll edit and suggests it inline. Accept with one click — no prompt needed.

Copilot Memory Preview

Persistent repo knowledge — coding conventions, patterns, preferences — across sessions. No more re-explaining context.

Roll back changes (CLI)

Rewind a session to a previous prompt; Copilot can restore files to their pre-edit state.

Q&A

Common questions

Is Copilot sending my code to the cloud?

Copilot uses code as context for suggestions. Enterprise plans include IP indemnity and data privacy controls. Check your org's policy.

How do I turn off inline suggestions?

Cmd+Shift+P → "Copilot: Toggle Inline Suggestions" or set "github.copilot.editor.enableAutoCompletions": false

What are the plan tiers?

7 tiers: Free · Student · Pro ($10) · Pro+ ($39) · Max ($100) · Business ($19/seat) · Enterprise ($39/seat)

Cloud agent requires paid plan. Claude & Codex require Pro+ or Enterprise. MCP requires admin enablement on Business/Enterprise.
